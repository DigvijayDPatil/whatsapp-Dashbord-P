PHASE 1.1 — SQLITE DATABASE SCHEMAS

Provisioning the SQLite Database & Models

Think of the database as a set of structured tables. In this dashboard, Django handles all SQLite model translations automatically using Python object-relational mapping (ORM).

How Models Look in Python code

Below is the core schema for the Tenant subscription table inside api/models.py:

api/models.py 
class Tenant(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    name = models.CharField(max_length=255)
    email = models.EmailField()
    license_key = models.CharField(max_length=100, unique=True)
    plan = models.CharField(max_length=100)
    created_at = models.DateTimeField(auto_now_add=True)
    expires_at = models.DateTimeField()
    status = models.CharField(max_length=50, default='active')
    waba_token = models.TextField(blank=True, default='')
    phone_number_id = models.CharField(max_length=100, blank=True, default='')
    template_name = models.CharField(max_length=150, default='missed_call_recovery')
    language_code = models.CharField(max_length=50, default='en_US')
    calls_count = models.IntegerField(default=0)
    messages_sent = models.IntegerField(default=0)

PHASE 3 — EXPIRY VALIDATION SCHEMES

Expiry Checking & License Validation Logic

When the webhook is triggered, the Django view first checks the tenant's license validity by parsing the active flag and verifying the expiry date in DB.

Secure Validation Architecture

All subscription parameters are isolated on the server. The client mobile app has no way to edit or skip the verification checks, preventing subscription spoofing.

validation_rules.py 
# Python validation logic
if tenant.status == 'blocked':
    return JsonResponse({'ok': False, 'error': 'Tenant blocked'}, status=403)

if tenant.expires_at < timezone.now():
    tenant.status = 'expired'
    tenant.save()
    return JsonResponse({'ok': False, 'error': 'License expired'}, status=403)

meta_api_helper.py 
import requests

def send_whatsapp_message(phone_id, token, to_phone, template, name):
    url = f"https://graph.facebook.com/v21.0/{phone_id}/messages"
    headers = {"Authorization": f"Bearer {token}", "Content-Type": "application/json"}
    payload = {
        "messaging_product": "whatsapp",
        "to": to_phone,
        "type": "template",
        "template": {
            "name": template,
            "language": {"code": "en_US"},
            "components": [{"type": "body", "parameters": [{"type": "text", "text": name}]}]
        }
    }
    return requests.post(url, json=payload, headers=headers)
